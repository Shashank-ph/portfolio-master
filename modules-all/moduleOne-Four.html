Unit 10 - Programming Outline

  • Initial Post

    Zero Trust Technology and Anti-Malware software.Safeguarding our systems and networks is crucial given the growing complexity of cyber threats.
    1. Introduction The goal of this assignment is to develop a simple Python-based solution that integrates Zero Trust Access Control and Anti-Malware mechanisms to enhance the security of an online shopping system as presented in Unit 9 previously. 2. Security Threats Addressed A. Unauthorized Access: o The Zero Trust Access Control mechanism ensures that only users with valid authentication tokens can access the system, minimizing the risk of unauthorized access. o Specific User roles are used to restrict access to certain parts of the system (e.g., admin panel, customer). B. Malware Infiltration: o The Anti-Malware scanner inspects files for known malicious patterns or signatures, preventing the upload or execution of infected files within the system. 3. Libraries and Tool Environment A. Standard Python Libraries: o The solution does not rely on any external libraries, focusing on using only default Python libraries for simplicity and portability. o If required, potential libraries can be included in future such as:  os and sys: For file operations and handling environment variables.  hashlib: For implementing more secure authentication tokens using hashing. B. Development Environment: o Python 3.12.4 version o Virtual Studio Code & Jupyter Notebook o Command line or terminal for running the Python script. 4. Solution Design A. Zero Trust Technology - Access Control: o Class Design: The Zero Trust Access Control class shall manage authentication and authorization.  Authentication: Verifies if the user has a valid token before allowing them to proceed.  Authorization: Checks if authenticated users have permission to access requested resources based on predefined user roles. B. Anti-Malware Mechanism: o Class Design: The Malware Scanner class shall scan files for known malware signatures.  Signature Detection: Compares the contents of files to a list of known malware patterns and alerts if malware is detected. 5. Test Plan 1. Objective: o To validate the effectiveness of the Zero Trust and Anti-Malware solutions by running a series of tests that simulate typical usage scenarios and potential security threats. 2. Testing Scenarios: o Zero Trust Access Control Testing: 1. Test Case 1: Attempt to authenticate a user with correct credentials.  Expected Outcome: User is successfully authenticated. 2. Test Case 2: Attempt to authenticate a user with incorrect credentials.  Expected Outcome: Authentication fails. 3. Test Case 3: Authenticate a user and check authorization to access permitted resources.  Expected Outcome: Access is granted. 4. Test Case 4: Authenticate a user and attempt to access restricted resources.  Expected Outcome: Access is denied. o Anti-Malware Scanner Testing: 1. Test Case 5: Scan a clean file that does not contain any malware signatures.  Expected Outcome: "File is clean" message is displayed. 2. Test Case 6: Scan a file that contains a known malware signature.  Expected Outcome: "Malware detected !" message is displayed. 3. Test Execution: o Using Visual Code output terminal to display results of each test. o Record screenshots of the output to demonstrate the success or failure of each test scenario. 6. Expected Output • Upon successful implementation and testing: o Users with valid credentials will be able to access only the resources they are permitted to use. o Any attempt to upload or execute a file with a known malware pattern will be flagged by the Anti-Malware system. o The solution should also demonstrate the ability to dynamically update malware definitions. 7. Documentation 1. README File: o Brief description of the implemented solution. o Step-by-step guide on how to set up, run, and test the program. o Other external sources or references used for the implementation (if any). 2. Source Code File/s: o Code file used to write and execute the program. o Explanation of key functions written in the source code itself. 3. Screenshots: o Expected output images of Zero Trust Technology & Anti-Malware Model as mentioned in above test cases. 8. Conclusion The implemented solution provides a basic yet effective approach in applying Zero Trust principles and malware detection to enhance security of an online grocery store environment.
